📖 Hướng dẫn đẩy báo cáo lên Google Drive

POS gửi file .xlsx bằng POST JSON tới Web App của bạn. Bạn cần một dự án Google Apps Script để nhận dữ liệu và tạo file trong thư mục Drive đã chọn.

  1. Tạo thư mục trên Drive và lấy ID
    Vào Google Drive → tạo thư mục (ví dụ Báo cáo POS) → mở thư mục → copy ID trên thanh địa chỉ:
    https://drive.google.com/drive/folders/CHÍNH_LÀ_ID_THƯ_MỤC
    Dán ID này vào ô ID thư mục Google Drive trong Cài đặt.
  2. Mở Google Apps Script và tạo dự án
    Truy cập script.google.com → Dự án mới → đặt tên (ví dụ POS Upload Báo cáo) → xóa code mặc định và dán toàn bộ mã mẫu ở bước 4.
  3. Lưu dự án
    Bấm Lưu (Ctrl+S), đặt tên nếu được hỏi.
  4. Dán mã Apps Script sau vào trình soạn thảo 
    function doPost(e) {
  try {
    var data = JSON.parse(e.postData.contents);
    var name = data.filename || 'bao_cao.xlsx';
    var mime = data.mimeType || 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet';
    var bytes = Utilities.base64Decode(data.base64Data);
    var blob = Utilities.newBlob(bytes, mime, name);
    var file;
    if (data.folderId && data.folderId.toString().trim() !== '') {
      file = DriveApp.getFolderById(data.folderId.toString().trim()).createFile(blob);
    } else {
      file = DriveApp.createFile(blob);
    }
    return ContentService
      .createTextOutput(JSON.stringify({ success: true, id: file.getId(), url: file.getUrl() }))
      .setMimeType(ContentService.MimeType.JSON);
  } catch (err) {
    return ContentService
      .createTextOutput(JSON.stringify({ success: false, error: err.toString() }))
      .setMimeType(ContentService.MimeType.JSON);
  }
}
    Mã này khớp định dạng POS gửi: filename, mimeType, base64Data, folderId. Ứng dụng gửi tiêu đề Content-Type: text/plain (thân vẫn là chuỗi JSON) để tránh lỗi CORS trên trình duyệt — bạn không cần sửa doPost.
  5. Triển khai Web App lần đầu (quan trọng)
    Bấm Triển khai → Quản lý Triển khai → biểu tượng Tạo triển khai → chọn loại Ứng dụng web → điền mô tả (tuỳ ý) → mục Thực thi dưới tên: chọn Tài khoản của tôi → Người có quyền truy cập: chọn Bất kỳ ai (để trình duyệt trên máy POS có thể gọi) hoặc Bất kỳ người dùng Google nếu bạn chỉ dùng nội bộ có đăng nhập Google.
    Bấm Triển khai → nếu hỏi Ủy quyền, chọn tài khoản Google và cho phép quyền Google Drive cho dự án.
  6. Lấy URL Web App
    Sau khi triển khai, copy URL Ứng dụng web — dạng https://script.google.com/macros/s/xxxxx/exec → dán vào ô URL Web App trong Cài đặt POS.
  7. Cập nhật mã (nếu sửa script)
    Mỗi lần sửa doPost, cần Triển khai → Quản lý Triển khai → chỉnh sửa biểu tượng bút → chọn Phiên bản mới → Triển khai. Nếu không, POS vẫn chạy bản cũ.
  8. Hoàn tất trong POS
    Điền đủ URL + ID thư mục → bấm Lưu cài đặt → dùng Đẩy báo cáo lên Drive hoặc Xuất Excel (nếu bật tự đẩy) để thử. File bao_cao_*.xlsx sẽ xuất hiện trong thư mục đã chọn.

Nếu báo “Failed to fetch”: dùng URL triển khai kết thúc /exec (không dùng /dev); triển khai lại Phiên bản mới sau mỗi lần sửa script; tắt extension chặn (uBlock, Privacy Badger…) đối với script.google.com; thử trình duyệt khác hoặc mạng khác. POS đã gửi text/plain và có chế độ dự phòng tương thích — nếu thành công nhưng không thấy file, kiểm tra đúng thư mục và quyền Drive.

Lưu ý bảo mật: Web App “Bất kỳ ai” có URL công khai — chỉ nên dùng URL khó đoán và không chia sẻ công khai. Ai biết URL có thể gửi POST (có thể thêm kiểm tra mật khẩu trong script nếu cần).
